## What I do
- Check for non-inclusive language in UI and docs
- Ensure diverse user scenarios are considered
- Validate error messages are respectful and helpful
- Review for cultural sensitivity

## When to use me
Use this skill when writing user-facing text, documentation, or API responses.

## Review Checklist
- [ ] Language is inclusive and respectful
- [ ] Error messages don't blame the user
- [ ] Documentation considers diverse backgrounds
- [ ] Examples represent diverse scenarios
